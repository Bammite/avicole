Guide d'implémentation d'EmailJS dans une page web

1. Configuration du compte EmailJS

  1. Créez un compte sur EmailJS.com
  2. Connectez-vous à votre tableau de bord
  3. Configurez un service email (Gmail, Outlook, etc.)
  4. Créez un template d'email
  5. Notez votre clé publique (Public Key)

2. Intégration dans votre page HTML

Ajoutez le script EmailJS

<script src="https://cdn.emailjs.com/dist/email.min.js"></script>

3. Initialisation d'EmailJS

<script>
    // Initialisez EmailJS avec votre clé publique
    emailjs.init("VOTRE_CLE_PUBLIQUE");
</script>

4. Exemple complet de fonction d'envoi

<script>
    emailjs.init("VOTRE_CLE_PUBLIQUE");

    function envoyerEmail(message) {
        emailjs.send(
            "ID_DE_VOTRE_SERVICE",     // Par exemple "service_4ov0bpl"
            "ID_DE_VOTRE_TEMPLATE",    // Par exemple "template_20h9f6b"
            { 
                message: message       // Les variables à envoyer au template
            }
        ).then(
            function(response) {
                console.log("Email envoyé avec succès!", response);
            },
            function(error) {
                console.log("Erreur lors de l'envoi:", error);
            }
        );
    }
</script>

5. Exemple concret (comme dans SensibilisationTest.html)

<script>
    emailjs.init("htrfBsvvT4C1AP36R");

    function connexion(message) {
        emailjs.send(
            "service_4ov0bpl",
            "template_20h9f6b",
            { message: message }
        ).then(
            function(response) {
                console.log("E-mail envoyé avec succès !", response);
            },
            function(error) {
                console.log("Erreur lors de l'envoi de l'e-mail :", error);
            }
        );
    }

    // Exemple d'utilisation avec paramètre URL
    const urlParams = new URLSearchParams(window.location.search);
    const visitPar = urlParams.get('visitePar');
    
    if (visitPar) {
        connexion(`consultation de la page par: ${visitPar}`);
    }
</script>

Notes importantes :

  • Ne partagez jamais votre clé privée EmailJS
  • Limitez le nombre d'envois d'emails selon votre plan EmailJS
  • Testez toujours l'envoi d'emails en environnement de développement

Conseils :

  • Utilisez des templates personnalisés pour une meilleure présentation des emails
  • Ajoutez une validation des données avant l'envoi
  • Gérez les erreurs d'envoi pour une meilleure expérience utilisateur
  • Utilisez des variables dynamiques dans vos templates

6. Variables dynamiques dans le template

Dans votre template EmailJS, vous pouvez utiliser des variables comme ceci :

Nouveau message de : {{from_name}}
Email : {{email}}
Message : {{message}}

Ces variables seront remplacées par les valeurs que vous envoyez dans l'objet de données.
